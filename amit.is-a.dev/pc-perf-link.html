Introduction

pc-perf-link is an open-source, ultra-lightweight hardware telemetry bridge. It completely isolates performance monitoring from your host machine by broadcasting real-time metrics (CPU, RAM, Network, Disk, and Top Apps) over a local network to any device with a web browser.

Sub-Second Latency

WebSocket powered push-architecture updates UI exactly every 1000ms.

Zero UI Overhead

Host runs purely in terminal/background. Graph rendering is offloaded to mobile.

Developer Grade

Built-in benchmarking tools specifically designed for C/C++/Rust optimization.

Why avoid 3rd Party Monitors?

Most commercial monitoring software (like NZXT CAM, Corsair iCUE, or web-based electron dashboards) suffer from severe architectural flaws when it comes to raw performance:

  • Electron Bloat: They run Chromium in the background, consuming 300MB - 600MB of RAM just to draw a graph.
  • CPU Interrupts: Polling and rendering complex UI on the host machine steals CPU cycles from your actual game or compilation tasks.
  • Data Privacy: Many require accounts and silently phone home with your telemetry data.

The pc-perf-link Solution

We utilize pure Python (`psutil`) to poll hardware at a near-native level. The backend consumes ~15MB of RAM. It bundles the data into JSON and fires it across your local WiFi. Your phone's processor handles all the heavy lifting of drawing charts.

Setup & Autostart

1

Install Dependencies

pip install fastapi uvicorn psutil
2

Manual Launch

To run it normally and see console logs (great for debugging):

python server_bridge.py
3

Enable Autostart (Background Mode)

You don't want a console window open all the time. We included `start_server.vbs` to execute the python script completely silently.

  1. Right-click start_server.vbs and select "Create shortcut".
  2. Press Win + R, type shell:startup, and hit Enter.
  3. Move the shortcut you created into that Startup folder.

How to stop it? Since it's hidden, open Windows Task Manager, find python.exe or pythonw.exe, and click "End Task".

Client & Kiosk Browser

Connecting via Browser

Open any device on your local WiFi and navigate to the IP address printed by the Python server (e.g., http://192.168.1.5:8000).

The web app features a Setup Screen. Enter the IP there; it is saved securely to your browser's localStorage so you never have to type it again.

The Kiosk APK (For old phones)

Don't throw away that old Android phone! Turn it into a permanent hardware monitor.

Data Architecture & WebSocket Schema

How Data Moves

Host PC psutil Hardware Poll
FastAPI Python Dict to JSON
WebSocket Push Client Browser Renders

Live JSON Payload

Updates 1/sec

Every 1000ms, the server pushes this complete state object. Notice the "apps" dictionary. The backend filters out processes using less than 50MB to ignore background services, returning only heavy applications grouped by their executable name.

websocket_payload.json 
{
  "ip": "192.168.1.5",
  "uptime": "2h 45m",
  "cpu": { "percent": 12.5, "freq_ghz": 3.8 },
  "ram": { "percent": 45, "used_gb": 7.2, "total_gb": 16.0 },
  "network": { "up_kbps": 120.5, "down_kbps": 2400.1 },
  "disk": { "read_kbps": 0.0, "write_kbps": 45.2 },
  "apps": {
    "count": 12,
    "top": [
      { "name": "chrome", "mem_mb": 1250.5 },
      { "name": "vscode", "mem_mb": 850.2 }
    ]
  }
}

Tap-to-Expand Modals

Tapping any specific card (CPU, RAM, Net, Disk) triggers a smooth micro-animation, pauses the main view, and opens a Full-Screen Interactive Chart maintaining a 60-second rolling history buffer directly in the browser's memory.

Active Apps Tracker

In the top navigation, tapping the APPS: 12 counter opens a horizontal bar chart displaying exact megabyte consumption of the top processes, updated dynamically.

Benchmarking Code (C/C++/Rust)

When writing low-level code where memory fragmentation and execution speed are critical, you need un-polluted data. The UI includes a massive red "RECORD" button specifically for tracking code execution limits.

The 3x2 Grid Report

Upon stopping a recording, the UI intercepts the data array and generates a comprehensive session report detailing:

  • CPU: Peak & Base-to-Max range
  • RAM: Total Delta & Peak Limit
  • Time: Millisecond accurate duration
  • Disk: Maximum I/O Read/Write spikes

Efficiency Benchmark Formula

Score = (Time × 50) + (Peak CPU) + (RAM_GB × 200)

A lower score indicates superior optimization. We penalize heavy RAM allocations heavily (200x multiplier) to encourage efficient memory management in unmanaged languages.

Frequently Asked Questions

Q Does this use a lot of PC resources?

No. Because the backend is pure Python generating JSON without rendering graphics, it utilizes roughly 15MB of RAM and less than 0.1% CPU overhead. The heavy lifting (rendering charts) is pushed entirely to your mobile device's GPU/CPU.

Q How do I find my PC's IP address?

When you run python server_bridge.py manually, the console will print a line saying Mobile URL: http://192.168.x.x:8000. Enter that exact IP into your phone.

Q Can I use it on an iPhone/iPad?

Yes. Simply open the URL in Safari, tap the Share button, and select "Add to Home Screen". This will run it as a standalone, full-screen Progressive Web App without browser UI interfering.

Q Why does my IP connection fail?

Ensure your PC and mobile device are on the same WiFi network. Additionally, your Windows Firewall might be blocking port 8000. You may need to allow Python through the firewall in your Windows Settings.

Q What happens if I close the browser tab?

The WebSocket connection is simply dropped. The Python server continues running silently in the background on your PC until you manually stop it. Your PC's performance remains completely unaffected by client disconnects.

Q How do I update pc-perf-link?

Updates can be seamlessly applied by downloading the latest server_bridge.py and index.html from the GitHub repository and overwriting your existing files.

Q Is it secure to broadcast telemetry over my network?

pc-perf-link is designed exclusively for Local Area Networks (LAN). Your data does not leave your router. However, anyone connected to your WiFi network can view the stats if they know the IP address.

Q Can I view the telemetry from outside my house?

Out-of-the-box, no. This tool prevents internet exposure for security. If you must view it remotely, you should set up a secure VPN (like Tailscale or ZeroTier) into your home network.

Q Does it support Linux or macOS?

Yes! Because the backend relies purely on the psutil Python library, it is completely cross-platform compatible and will run perfectly fine on Linux and macOS environments.

Q Why is the Top Apps section not showing my game?

The backend actively filters out any process consuming less than 50MB of RAM to prevent UI clutter. It also matches executables; ensure your game's memory threshold exceeds this limit and isn't hidden by system anti-cheat obfuscation.

How to Contribute

We actively welcome contributions from the community! Whether it's a bug fix, new feature, or documentation improvement, here is the standard pipeline to get your code merged into pc-perf-link.

1

Fork & Clone

Start by forking the repository to your own GitHub account, then clone it locally and create a new branch for your feature or fix.

git clone https://github.com/YOUR_USERNAME/pc-perf-link.git
cd pc-perf-link
git checkout -b feature/my-awesome-feature
2

Commit & Push

Make your code changes, ensuring you maintain the low-memory, zero-bloat philosophy. Then commit your changes with a clear message and push to your fork.

git add .
git commit -m "feat: added new awesome feature"
git push origin feature/my-awesome-feature
3

Open PR & Email Me

Go to the original GitHub repository and click "Compare & pull request". Fill out the PR template describing your changes.
